Приложение accounts
Назначение

Отвечает за пользователей системы и их роли.

Основная модель
User

Кастомная модель пользователя (наследник AbstractUser).

Хранит:

стандартные поля Django (username, password, email и т.д.)

role — роль пользователя:

student

teacher

Используется для:

разграничения логики (кто студент, кто учитель)

фильтрации данных в учебных моделях




Приложение learning
Назначение

Отвечает за учебный контент, выполнение заданий студентами и хранение результатов.

Контент
Topic

Тема обучения.

Хранит:

title — название темы

description — описание

is_active — активность темы

Используется для:

группировки вопросов

запуска попыток по теме

Question

Вопрос из банка вопросов.

Хранит:

topic — тема, к которой относится вопрос

text — текст вопроса

question_type — тип вопроса (сейчас только multiple_choice)

is_active — активность вопроса

created_at — дата создания

Используется как шаблон вопроса, который может быть выдан многократно.

Choice

Вариант ответа для вопроса.

Хранит:

question — к какому вопросу относится

text — текст варианта

is_correct — является ли вариант правильным

order — порядок отображения

Используется для формирования multiple choice вопросов.

Выполнение заданий
Attempt

Попытка студента выполнить задание по теме.

Хранит:

student — пользователь с ролью student

topic — тема попытки

started_at — время начала

finished_at — время завершения (может быть пустым)

status — состояние попытки (in_progress, completed, abandoned)

Используется для фиксации факта выполнения задания и истории попыток.

AttemptQuestion

Связь между попыткой и вопросом.

Хранит:

attempt — попытка

question — выданный вопрос

order — порядковый номер вопроса в попытке

Фиксирует, какие именно вопросы были выданы в конкретной попытке.

Answer

Ответ студента на вопрос в рамках попытки.

Хранит:

attempt_question — на какой вопрос и в какой попытке дан ответ

selected_choices — выбранные варианты ответа

answered_at — время ответа

is_correct — корректность ответа (или null)

teacher_comment — комментарий учителя

Используется для хранения результатов и проверки ответов.

Организация студентов
Group

Учебная группа.

Хранит:

name — название группы

description — описание

teacher — ответственный учитель

is_active — активность группы

Используется для логической группировки студентов.
